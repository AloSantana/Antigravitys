## Color Scheme

### Dark Theme Colors
```
Background:     #0f172a (Primary)
                #1e293b (Secondary)
                #334155 (Tertiary)

Text:           #f8fafc (Primary)
                #94a3b8 (Secondary)
                #64748b (Muted)

Accents:        #3b82f6 (Blue)
                #10b981 (Green)
                #8b5cf6 (Purple)
                #f59e0b (Orange)

Status:         #34d399 (Success)
                #f87171 (Error)
                #fbbf24 (Warning)
```

### UI Elements

**Buttons**
- Primary: Blue → Purple gradient
- Secondary: Gray with border
- Success: Green gradient
- Danger: Red

**Cards**
- Background: Secondary color
- Border: Subtle white (8% opacity)
- Hover: Transform + border color change

**Status Indicators**
- Active: Green with pulse animation
- Inactive: Gray
- Error: Red

**Code Display**
- Background: #1e1e1e (VSCode-like)
- Text: #d4d4d4
- stdout: Green (#4ec9b0)
- stderr: Red (#f48771)

## Interactive Elements

### Hover Effects
```
Cards/Items:    → Transform translateY(-2px)
                → Border color change
                → Box shadow

Buttons:        → Transform translateY(-2px)
                → Box shadow increase
                → Brightness increase

Tabs:           → Background change
                → Text color change
```

### Loading States
```
Spinner:        → Rotating circle animation
                → Blue accent color
                → Centered with text

Buttons:        → "⏳ Loading..." text
                → Disabled state
                → No hover effects
```

### Focus States
```
Inputs:         → Blue border
                → Subtle blue shadow (glow)
                → Outline removed
```

## Accessibility Features

✅ **Semantic HTML**: Proper heading hierarchy  
✅ **Labels**: All inputs have labels  
✅ **ARIA**: Status indicators for screen readers  
✅ **Keyboard**: Tab navigation works  
✅ **Contrast**: Text meets WCAG AA standards

## Mobile Experience

### Touch Targets
- Minimum 44x44px touch areas
- Adequate spacing between interactive elements
- No hover-dependent functionality

### Layout Adjustments
- Full-width content on mobile
- Single-column grids
- Larger text for readability
- Collapsible sections

### Performance
- Lazy loading of tab content
- Efficient DOM updates
- Minimal re-renders
